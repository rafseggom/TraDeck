\subsection{Orquestación del Despliegue con Hardhat Ignition}
El paso a producción (o a redes de prueba) de una arquitectura multi-contrato requiere precisión para 
mantener las referencias entre ellos. En lugar de utilizar \textit{scripts} de despliegue imperativos, 
se optó por \textbf{Hardhat Ignition}.

Ignition proporciona un enfoque declarativo, permitiendo definir módulos de despliegue. El motor se 
encarga automáticamente de:
\begin{enumerate}
    \item Calcular los costes de Gas óptimos para la transacción de creación del contrato.
    \item Firmar las transacciones con las claves privadas configuradas de forma segura.
    \item La resolución de dependencias entre \texttt{TraDeckCoin} y \texttt{TraDeckNFT}.
    \item Generar los artefactos (ABIs) necesarios para la integración con la interfaz de usuario.
\end{enumerate}
